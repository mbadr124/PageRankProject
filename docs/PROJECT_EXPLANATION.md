# Project Explanation

This project implements the PageRank algorithm for a CSE455 High Performance Computing assignment. PageRank assigns an importance score to each node in a directed graph by repeatedly distributing rank through outgoing links until the scores converge.

## Main Idea

The graph is stored using CSR, or Compressed Sparse Row, format:

- `row_ptr` marks where each node's outgoing neighbors start and end.
- `col_idx` stores the destination nodes for all edges.
- `out_degree` stores how many outgoing links each node has.

Before running PageRank, the code builds an incoming-edge CSR version of the graph with `buildInCSR`. This makes each node update efficient, because PageRank needs to sum rank contributions from nodes that link into it.

Each iteration:

1. Starts from the current rank vector.
2. Computes the total rank held by dangling nodes, which are nodes with no outgoing edges.
3. Applies the PageRank formula using damping factor `0.85`.
4. Measures the L1 difference between old and new ranks.
5. Stops when the difference is below `1e-6`, or when `MAX_ITERATIONS` is reached.

The final rank values should sum to about `1.0`.

## Implementations

`src/Sequential.cpp` is the baseline single-threaded implementation. It updates every node one after another and is used as the reference for correctness and performance comparison.

`src/SharedMemory.cpp` uses OpenMP for shared-memory parallelism. It parallelizes the dangling-node sum, node rank updates, and convergence difference calculation. It uses double buffering with `rank` and `new_rank`, so threads read from one vector and write to separate indexes in the other.

`src/MessagePassing.cpp` uses MPI for distributed-memory parallelism. Rank 0 builds the graph and broadcasts it to all processes. The nodes are split into contiguous blocks, each process updates its own block, then processes exchange the updated rank values through MPI communication. Global sums and convergence checks use MPI reductions.

## Input Graphs

Each implementation can run with either:

- A random graph generated by `buildRandomGraph`.
- A graph loaded from an edge-list file in `data/`.

The current code uses the predefined graph, which contains `2000` nodes. The executable graph data is stored in `data/predefined_graph_edges.txt`, while `data/graph2_2000nodes.gexf` is included for visualizing the graph in tools such as Gephi. For testing, `data/tiny_graph_edges.txt` is useful for manual debugging and `data/graph_50k_edges.txt` is useful for OpenMP and MPI benchmarking.

## Purpose

The project compares three ways to run the same PageRank workload:

- Sequential execution on one CPU thread.
- Shared-memory parallel execution with OpenMP.
- Message-passing parallel execution with MPI.

This makes it useful for studying speedup, parallel overhead, synchronization, communication cost, and how graph algorithms behave on different HPC programming models.
